# MoaDay 요구사항 명세서

## 1. 문서 정보

| 항목 | 내용 |
| --- | --- |
| 제품명 | MoaDay |
| 제품 형태 | 가족·친구용 멀티스페이스 협업 SaaS의 로컬 MVP |
| 클라이언트 | 반응형 웹, 향후 iOS·Android 앱 고려 |
| 기준 구현 | 저장소의 현재 작업 트리와 DB 마이그레이션 V1~V14 |
| 상태 표기 | 구현, 부분 구현, 예정 |

이 문서는 제품 아이디어가 아니라 현재 구현과 검증 가능한 동작을 기준으로 작성했습니다.

## 2. 범위

### 포함

- 이메일 기반 계정과 개인 공간
- 가족·친구 그룹 공간과 역할 기반 권한
- 이메일·앱 내 초대 수락과 거절
- 캘린더, 반복 일정, 참석, 리마인더, ICS
- 게시글, 첨부파일, 댓글, 태그와 검색
- 이미지 또는 바코드 쿠폰, 선점과 사용 이력
- 앱 내 알림과 선택적 이메일 알림
- 통합검색, 오늘 대시보드와 감사 로그
- Docker 로컬 실행, 백업·복원, 임시 터널

### 제외 또는 예정

- 상용 클라우드 배포와 고가용성
- 결제·구독·사용량 제한
- 소셜 로그인, 이메일 인증, 비밀번호 재설정
- Google Docs 수준 실시간 공동 편집
- 네이티브 iOS·Android 앱과 푸시 알림
- 바이러스 검사, S3 호환 오브젝트 스토리지

## 3. 사용자와 역할

### 시스템 사용자

- 비회원: 회원가입, 로그인, 초대 링크 진입
- 회원: 개인 공간과 참여 중인 그룹 공간 사용

### 공간 역할

| 기능 | OWNER | ADMIN | MEMBER | VIEWER |
| --- | :---: | :---: | :---: | :---: |
| 공간 조회 | O | O | O | O |
| 일정·게시글·쿠폰 조회 | O | O | O | O |
| 일정·게시글·댓글 작성 | O | O | O | X |
| 쿠폰 등록·선점 | O | O | O | X |
| 게시글 고정 | O | O | X | X |
| 구성원 초대·추방 | O | O | X | X |
| ADMIN 지정·관리 | O | X | X | X |
| 공간 삭제 | O | X | X | X |
| 공간 탈퇴 | X | O | O | O |

소유자는 자신을 추방하거나 일반 탈퇴할 수 없고, 개인 공간은 삭제하거나 탈퇴할 수 없습니다.

## 4. 기능 요구사항

### 4.1 인증과 계정

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| AUTH-01 | 사용자는 이메일, 8~72자 비밀번호, 이름, 시간대로 가입한다. | 구현 | 중복 이메일은 409, 성공은 JWT와 사용자 정보를 반환한다. |
| AUTH-02 | 가입 시 개인 공간과 OWNER 멤버십을 자동 생성한다. | 구현 | 가입 직후 공간 목록에 개인 공간이 1개 이상 존재한다. |
| AUTH-03 | 사용자는 이메일·비밀번호로 로그인한다. | 구현 | 올바른 자격 증명은 1시간 JWT, 잘못된 값은 401을 반환한다. |
| AUTH-04 | 비밀번호는 평문이 아닌 적응형 해시로 저장한다. | 구현 | Spring Security 위임형 PasswordEncoder를 사용한다. |
| AUTH-05 | 웹은 새로고침 후 유효한 로컬 세션을 복원한다. | 구현 | 만료 전 토큰으로 다시 API를 사용할 수 있다. |
| AUTH-06 | 사용자는 현재 비밀번호 확인 후 계정을 삭제한다. | 구현 | 계정 상태와 소유 공간을 정리하고 이후 로그인을 차단한다. |
| AUTH-07 | 이메일 비밀번호 재설정을 제공한다. | 구현 | 계정 존재 여부를 노출하지 않고 30분 유효 일회용 토큰을 Outbox 메일로 발송한다. |
| AUTH-08 | 반복 로그인 실패와 비밀번호 변경 이후 세션을 보호한다. | 구현 | 5회 실패 시 15분 잠그고 비밀번호 변경·복구 시 기존 JWT 보안 버전을 무효화한다. |
| AUTH-09 | 가입 이메일을 별도로 인증한다. | 예정 | 인증 완료 전 외부 공유 기능을 제한한다. |

### 4.2 공간과 구성원

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| SPACE-01 | 회원은 가족 또는 친구 공간을 생성한다. | 구현 | 공간과 OWNER 멤버십을 만들고 첫 캘린더 조회 시 기본 캘린더를 보장한다. |
| SPACE-02 | 참여 중인 활성 공간만 목록에 표시한다. | 구현 | 탈퇴·추방·삭제된 공간은 목록에서 제외된다. |
| SPACE-03 | 관리자는 구성원과 역할을 확인한다. | 구현 | 활성 구성원의 이름, 이메일, 역할, 가입 시각을 제공한다. |
| SPACE-04 | 권한 있는 사용자는 역할을 변경하거나 구성원을 추방한다. | 구현 | OWNER 보호와 ADMIN 상호 관리 제한을 적용한다. |
| SPACE-05 | 일반 구성원은 공간에서 탈퇴한다. | 구현 | 멤버십이 즉시 비활성화되어 접근할 수 없다. |
| SPACE-06 | OWNER는 그룹 공간을 삭제한다. | 구현 | 초대와 멤버십을 종료하고 공간을 보관 상태로 변경한다. |
| SPACE-07 | 공간 소유권을 다른 ADMIN에게 이전한다. | 예정 | 이전 확인 후 기존 OWNER는 ADMIN으로 변경할 수 있다. |

### 4.3 초대

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| INV-01 | OWNER·ADMIN은 이메일과 역할로 구성원을 초대한다. | 구현 | 7일 유효 일회용 토큰과 초대 레코드를 만든다. |
| INV-02 | 초대 대상에게 수락 링크가 포함된 메일을 보낸다. | 구현 | 업무 트랜잭션에서 Outbox에 저장하고 별도 작업자가 SMTP로 발송한다. |
| INV-03 | 기존 회원은 받은 초대 목록을 확인한다. | 구현 | 로그인 이메일과 일치하는 유효한 초대만 표시한다. |
| INV-04 | 대상 회원은 초대를 수락하거나 거절한다. | 구현 | 수락 시 멤버십을 만들고 거절 시 다시 대기 목록에 나오지 않는다. |
| INV-05 | 관리자는 대기 초대를 취소하고 만료 상태를 확인한다. | 구현 | 취소·수락·거절·만료 상태를 구분한다. |
| INV-06 | 이미 활성 멤버이거나 동일한 대기 초대가 있으면 중복을 차단한다. | 구현 | 409 오류를 반환한다. |
| INV-07 | 관리자는 대기 중인 초대 메일을 재발송한다. | 구현 | 기존 토큰을 무효화하고 새 토큰·Outbox·감사 기록을 만든다. |

### 4.4 캘린더와 일정

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| CAL-01 | 공간마다 여러 캘린더를 생성·수정·삭제한다. | 구현 | 이름 중복을 차단하고 역할 권한을 적용한다. |
| CAL-02 | 월·주·일 화면으로 기간 일정을 조회한다. | 구현 | 지정 `from`~`to` 구간의 발생 일정을 반환한다. |
| CAL-03 | 일정 제목, 시간, 시간대, 장소, 설명, 종일 여부를 관리한다. | 구현 | 종료가 시작보다 뒤이고 IANA 시간대가 유효해야 한다. |
| CAL-04 | 매일·매주·매월·매년 반복과 종료일을 지원한다. | 구현 | 조회 구간에서 반복 발생 시각을 계산한다. |
| CAL-05 | 특정 반복 회차만 수정·취소·복원한다. | 구현 | 원본 규칙을 유지하고 회차 예외로 처리한다. |
| CAL-06 | 참석자를 지정하고 참석·불참·미정으로 응답한다. | 구현 | 같은 공간의 활성 구성원만 참석자가 될 수 있다. |
| CAL-07 | 일정별 0분~7일 전 리마인더를 최대 5개 설정한다. | 구현 | 중복 발생·사용자별 전달을 방지한다. |
| CAL-08 | 공간 일정을 ICS로 내보내고 캘린더로 가져온다. | 구현 | 최대 2MB·200개 VEVENT, UID 중복을 건너뛴다. |
| CAL-09 | 일정에 같은 공간의 게시글·첨부파일·쿠폰을 연결한다. | 구현 | 최대 50개, 다른 공간 리소스는 거부한다. |

### 4.5 공유함

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| POST-01 | 작성 권한 사용자는 게시글을 생성·수정·삭제한다. | 구현 | 작성자 또는 관리 권한에 따라 변경을 허용한다. |
| POST-02 | 관리자는 게시글을 고정한다. | 구현 | 고정 글이 목록 상단에 표시된다. |
| POST-03 | 게시글에 태그를 최대 10개 저장한다. | 구현 | `#` 제거, 소문자 정규화, 태그당 최대 40자를 적용한다. |
| POST-04 | 제목·본문·태그로 공간 게시글을 검색한다. | 구현 | 공간 경계를 벗어난 결과를 반환하지 않는다. |
| POST-05 | 댓글을 생성·수정·삭제한다. | 구현 | VIEWER 작성을 차단하고 작성자·관리자만 변경한다. |
| POST-06 | 게시글당 최대 20개, 파일당 20MB 첨부를 허용한다. | 구현 | 위험 형식·확장자·실행 시그니처를 거부한다. |
| POST-07 | 문서를 실시간 공동 편집한다. | 제외 | MVP는 게시글·첨부파일·댓글 방식으로 대체한다. |

### 4.6 쿠폰함

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| COUPON-01 | 쿠폰명, 브랜드, 만료일, 설명을 등록한다. | 구현 | 만료일은 미래여야 한다. |
| COUPON-02 | 원본 이미지를 저장하거나 바코드를 선택적으로 입력한다. | 구현 | JPG·PNG·WebP·GIF·AVIF 이미지, CODE128·EAN13을 지원한다. |
| COUPON-03 | 이미지가 있으면 인증된 구성원에게 확대 보기로 제공한다. | 구현 | 공개 정적 URL 없이 인증 API로 읽는다. |
| COUPON-04 | 구성원은 사용 가능한 쿠폰을 15분간 선점한다. | 구현 | 동시 요청 중 한 명만 성공하고 만료 시 자동 해제한다. |
| COUPON-05 | 선점자만 바코드를 조회하고 사용 완료·해제한다. | 구현 | 미선점 사용자에게 바코드 값을 반환하지 않는다. |
| COUPON-06 | 만료일이 지나면 자동 만료하고 알림·감사 로그를 남긴다. | 구현 | AVAILABLE·CLAIMED만 EXPIRED로 전환한다. |
| COUPON-07 | OWNER·ADMIN 또는 쿠폰 소유자는 사유를 입력해 상태를 정정한다. | 구현 | 최소 5자 사유와 변경 전후 상태를 기록한다. |

### 4.7 알림, 검색과 운영

| ID | 요구사항 | 상태 | 수용 기준 |
| --- | --- | --- | --- |
| NOTI-01 | 사용자는 최근 100개 앱 내 알림을 확인하고 읽음 처리한다. | 구현 | 읽지 않은 개수를 상단 배지에 표시한다. |
| NOTI-02 | 일정·게시글·쿠폰 활동별 알림 설정을 저장한다. | 구현 | 비활성 카테고리는 새 알림을 만들지 않는다. |
| NOTI-03 | 이메일 알림을 설정된 SMTP 발신 주소로 전송한다. | 구현 | 초대·일반 활동 메일이 공통 발송 계층과 발신 주소를 사용한다. |
| NOTI-04 | 이메일을 Outbox로 비동기 발송하고 실패를 자동 재시도한다. | 구현 | 최대 시도 횟수, 다음 시도 시각, 오류와 성공 시각을 저장하고 관리자가 최근 100건을 확인한다. |
| DISC-01 | 모든 참여 공간의 일정·게시글·쿠폰을 통합검색한다. | 구현 | 사용자 멤버십이 있는 데이터만 반환한다. |
| DISC-02 | 오늘 화면에 공간 수, 예정 일정, 임박 쿠폰, 미확인 알림을 표시한다. | 구현 | 실제 사용자 데이터를 집계한다. |
| DISC-03 | 알림·검색 결과에서 대상 상세 화면을 바로 연다. | 구현 | URL의 공간·대상 상태를 복원해 일정·공유글·쿠폰 상세를 연다. |
| OPS-01 | Docker Compose 한 명령으로 전체 서비스를 실행한다. | 구현 | 웹·API·DB·메일·게이트웨이 상태를 확인할 수 있다. |
| OPS-02 | DB와 업로드 파일을 함께 백업·복원한다. | 구현 | 매니페스트와 두 저장소를 일관된 시점에 보존한다. |
| OPS-03 | 임시 HTTPS 터널로 다른 디바이스에서 접속한다. | 구현 | 선택 프로필로 Quick Tunnel을 시작·종료한다. |

## 5. 비기능 요구사항

| ID | 분류 | 요구사항 | 현재 대응 |
| --- | --- | --- | --- |
| NFR-SEC-01 | 인증 | 보호 API는 유효한 JWT를 요구한다. | Spring Security Resource Server, stateless 세션 |
| NFR-SEC-02 | 테넌트 격리 | 모든 공간 자원 접근에서 활성 멤버십을 검증한다. | 서비스 계층 권한 검사와 통합 테스트 |
| NFR-SEC-03 | 비밀정보 | 비밀번호·토큰·SMTP 자격 증명을 저장소에 커밋하지 않는다. | `.env`와 로그·백업 경로 Git 제외 |
| NFR-SEC-04 | 파일 | 경로 순회와 실행 가능한 첨부를 차단한다. | 임의 저장 키, 정규화 경로, 형식·내용 검사 |
| NFR-SEC-05 | 계정 복구 | 복구 토큰 원문과 계정 존재 여부를 노출하지 않는다. | SHA-256 토큰 해시, 공통 응답, 단일 사용·만료·재요청 제한 |
| NFR-DATA-01 | 일관성 | 스키마 변경을 순차 적용하고 엔티티와 일치시킨다. | Flyway V1~V14, `ddl-auto=validate` |
| NFR-CON-01 | 동시성 | 쿠폰 중복 선점을 방지한다. | DB 비관적 쓰기 잠금과 트랜잭션 |

## 6. 주요 업무 규칙

1. 이메일은 공백 제거 후 소문자로 정규화합니다.
2. 초대는 생성 시점부터 7일간 유효하며 이메일이 일치하는 계정만 수락합니다.
3. 개인 공간은 가입 시 자동 생성되며 수동 생성·삭제·탈퇴할 수 없습니다.
4. VIEWER는 조회 중심 역할이며 게시글·댓글·일정 변경이 제한됩니다.
5. 쿠폰은 `AVAILABLE → CLAIMED → USED` 또는 `AVAILABLE/CLAIMED → EXPIRED`로 전이합니다.
6. 쿠폰 선점은 기본 15분이며 같은 시점에 한 사용자만 성공합니다.
7. 바코드는 쿠폰에 선택 사항이지만 값이 있으면 형식 검증을 통과해야 합니다.
8. 일정 연결 자료는 반드시 일정과 같은 공간에 있어야 합니다.
9. 반복 일정 회차 예외는 원래 시작 시각과 이벤트 ID 조합으로 하나만 존재합니다.
10. 삭제가 감사·참조에 영향을 주는 자원은 가능한 경우 상태값으로 비활성화합니다.
