# MoaDay MVP 제품 요구사항(PRD)

작성일: 2026-07-13  
대상: 가족·친구 소그룹  
플랫폼: 반응형 웹/PWA

## 1. 제품 목표

가족과 친구가 흩어진 일정, 공지·파일, 모바일 쿠폰을 한 그룹에서 확인하고 중복 사용 없이 함께 관리하게 한다.

MVP가 검증해야 할 가설은 세 가지다.

1. 공유 캘린더 사용자에게 일정과 연결된 글·파일을 같은 화면에서 보는 것이 유용하다.
2. 개인이 가진 쿠폰을 신뢰 관계의 그룹에 공유할 의향이 있다.
3. 쿠폰 선점과 사용 이력이 중복 사용에 대한 불안을 충분히 낮춘다.

## 2. 성공 지표

베타 4주 기준:

- 신규 가입자의 60% 이상이 24시간 안에 그룹을 생성하거나 초대를 수락
- 생성된 그룹의 50% 이상이 구성원 2명 이상 확보
- 활성 그룹의 50% 이상이 한 주에 일정 또는 게시글을 2건 이상 생성
- 등록 쿠폰의 30% 이상이 만료 전에 선점 또는 사용 완료
- 같은 쿠폰에 대한 동시 선점 성공은 항상 1건
- 주간 활성 그룹의 4주차 유지율 35% 이상
- 권한 오류 및 다른 그룹 데이터 노출 사고 0건

수치는 제품 가설 검증을 위한 초기 목표이며 베타 표본 크기와 사용 빈도에 따라 조정한다.

## 3. MVP 사용자와 주요 상황

### 가족 관리자

- 가족 캘린더와 쿠폰함을 만들고 구성원을 초대한다.
- 병원 예약, 학교 행사, 여행 일정을 구성원별 색상으로 확인한다.
- 여행 계획 글과 예약 확인서를 일정에 연결한다.
- 만료가 가까운 쿠폰을 가족에게 공유한다.

### 가족/친구 구성원

- 받은 초대를 수락하고 오늘 해야 할 일을 확인한다.
- 공유 일정에 참석 여부를 답한다.
- 공지 글에 댓글을 남기고 파일을 내려받는다.
- 쿠폰을 선점한 후 바코드를 열어 사용하고 완료 처리한다.

### 친구 모임장

- 모임 일정을 만들고 장소 링크와 준비사항을 공유한다.
- 모임에서 쓸 쿠폰을 일정과 연결한다.
- 참석 여부와 쿠폰 사용 상태를 한 화면에서 확인한다.

## 4. 화면 구조

| 화면 | 목적 | 핵심 행동 |
|---|---|---|
| 시작/로그인 | 계정 접근 | 이메일 또는 소셜 로그인 |
| 첫 설정 | 개인 공간과 첫 그룹 준비 | 이름·시간대 설정, 그룹 만들기/초대 수락 |
| 오늘 | 모든 중요 정보를 요약 | 오늘 일정, 임박 쿠폰, 최근 글 확인 |
| 캘린더 | 개인·그룹 일정 관리 | 필터, 일정 생성/편집, 참석 응답 |
| 일정 상세 | 일정에 필요한 자료 결합 | 참석, 댓글이 아닌 연결 글/파일/쿠폰 확인 |
| 공유함 | 공지와 자료 축적 | 글 작성, 파일 첨부, 댓글, 검색 |
| 쿠폰함 | 공유 쿠폰 상태 관리 | 등록, 선점, 열람, 완료/해제 |
| 공간 관리 | 구성원과 권한 관리 | 초대, 역할, 내보내기, 알림 설정 |
| 내 설정 | 개인 환경 관리 | 프로필, 시간대, 알림, 계정 삭제 |

모바일 하단 내비게이션은 `오늘 / 캘린더 / 공유함 / 쿠폰함` 4개로 제한하고 공간 전환과 설정은 상단 메뉴에 둔다.

## 5. 공통 UX 원칙

- 사용자가 현재 보고 있는 데이터의 공간을 항상 색상과 이름으로 표시한다.
- 개인 일정과 그룹 일정이 섞여도 공개 범위가 화면에서 구분되어야 한다.
- 쿠폰 번호와 바코드는 명시적인 선점과 재확인 전에는 표시하지 않는다.
- 삭제·구성원 제거·쿠폰 상태 정정은 결과와 영향을 먼저 설명한다.
- 색상만으로 공간, 참석 상태, 쿠폰 상태를 구분하지 않고 텍스트/아이콘을 함께 쓴다.
- 모바일 320px 너비부터 데스크톱까지 핵심 행동이 가려지지 않아야 한다.
- 빈 화면에는 설명보다 첫 행동 하나를 명확하게 제시한다.

## 6. 에픽 A — 인증과 온보딩

### A-01 계정 생성/로그인

사용자는 이메일 또는 우선 선정된 소셜 공급자로 로그인할 수 있다.

인수 조건:

- 인증 성공 시 개인 공간과 기본 캘린더가 없으면 자동 생성한다.
- 로그인 실패는 계정 존재 여부를 과도하게 노출하지 않는 동일 계열 메시지로 표시한다.
- 로그아웃 시 서버 세션 또는 갱신 토큰을 폐기한다.
- 탈퇴 사용자의 이메일 재사용 정책을 명확히 적용한다.

### A-02 첫 설정

인수 조건:

- 표시 이름과 IANA 시간대를 저장한다.
- 사용자는 `새 그룹 만들기`와 `받은 초대 확인` 중 선택할 수 있다.
- 건너뛴 사용자는 오늘 화면에서 언제든 그룹을 만들 수 있다.

## 7. 에픽 B — 공간과 초대

### B-01 그룹 생성

인수 조건:

- 그룹 유형은 가족 또는 친구 중 선택한다.
- 그룹 이름, 대표 색상, 기본 시간대를 지정한다.
- 생성자는 소유자가 된다.
- 무료 플랜 한도를 넘으면 생성 전 안내한다.

### B-02 구성원 초대

인수 조건:

- 소유자/관리자는 이메일 초대 또는 만료되는 초대 링크를 만들 수 있다.
- 초대에는 그룹 이름, 초대한 사람, 부여될 역할, 만료 시각이 표시된다.
- 취소·만료·이미 사용된 초대는 수락할 수 없다.
- 동일 이메일에 활성 초대를 중복 발송하지 않고 기존 초대 재전송을 제안한다.
- 초대 링크 원문은 서버 로그와 DB에 저장하지 않는다.

### B-03 구성원 관리

인수 조건:

- 소유자는 관리자/멤버/열람자로 역할을 바꿀 수 있다.
- 마지막 소유자는 탈퇴하거나 제거될 수 없다.
- 제거된 사용자는 즉시 새 API 요청과 파일/쿠폰 URL 접근이 거부된다.
- 구성원/역할 변경은 감사 이력에 남는다.

## 8. 에픽 C — 오늘 화면

### C-01 통합 요약

인수 조건:

- 접근 가능한 개인·그룹 공간의 오늘 일정과 다음 일정을 시간순으로 표시한다.
- 7일 이내 만료되는 사용 가능 쿠폰을 표시한다.
- 최근 고정 글과 읽지 않은 알림을 표시한다.
- 공간, 캘린더, 구성원별 필터를 제공하되 마지막 선택을 사용자 설정으로 기억한다.
- 권한이 `바쁨만`인 일정은 시간 블록 외 정보를 표시하지 않는다.

## 9. 에픽 D — 캘린더

### D-01 캘린더 관리

인수 조건:

- 개인/그룹 공간에 여러 캘린더를 만들 수 있다.
- 이름과 색상을 수정하고 표시/숨김 상태를 저장한다.
- 캘린더 삭제 전에 포함 일정 수와 영향을 안내한다.

### D-02 일정 생성/수정

필수 필드: 캘린더, 제목, 시작, 종료.  
선택 필드: 종일, 시간대, 장소, URL, 설명, 참석자, 공개 범위, 반복, 알림.

인수 조건:

- 종료는 시작보다 빠를 수 없다.
- 그룹 일정 기본 공개 범위는 상세 공개다.
- 개인 일정을 그룹 통합 화면에 표시할 때 상세/제목만/바쁨만 중 선택할 수 있다.
- 반복 일정은 매일/매주/매월/매년과 종료일/횟수를 지원한다.
- 반복 일정 수정 시 이번 일정/이후 일정/전체 일정을 반드시 선택한다.
- 버전 충돌 시 다른 사용자의 변경을 덮지 않고 새로고침/재적용 선택지를 보여준다.

### D-03 참석 응답과 연결 자료

인수 조건:

- 참석자는 참석/미정/불참으로 응답할 수 있다.
- 일정 작성자는 응답 현황을 확인한다.
- 일정에는 같은 공간의 글·파일·쿠폰을 연결할 수 있다.
- 쿠폰 연결 카드에서는 상태와 만료일만 표시하고 바코드는 표시하지 않는다.

## 10. 에픽 E — 공유함

### E-01 게시글

인수 조건:

- 제목과 리치 텍스트 본문을 작성한다.
- 링크, 이미지, 허용된 파일 형식을 첨부한다.
- 작성자/관리자는 수정하고 작성자/관리자만 삭제할 수 있다.
- 수정 시 버전과 편집자를 기록한다.
- 관리자는 중요한 글을 고정할 수 있다.
- 실시간 공동 편집과 오프라인 편집 병합은 제공하지 않는다.

### E-02 댓글

인수 조건:

- 멤버는 댓글을 작성하고 자신의 댓글을 수정/삭제할 수 있다.
- 관리자는 부적절한 댓글을 삭제할 수 있고 감사 이력을 남긴다.
- 글 작성자와 언급된 구성원에게 알림을 보낸다.

### E-03 검색

인수 조건:

- 글 제목과 본문, 일정 제목, 쿠폰 브랜드/상품명을 검색한다.
- 결과는 권한이 있는 공간 안에서만 반환한다.
- 쿠폰 번호/바코드, 초대 이메일은 검색하지 않는다.

## 11. 에픽 F — 쿠폰함

### F-01 쿠폰 등록

필수 필드: 대상 그룹, 브랜드, 상품명, 유효기간, 쿠폰 이미지 또는 번호.  
선택 필드: 메모, 사용 조건, 연결 일정.

인수 조건:

- 등록 전 공유 가능 여부와 발행사 약관 확인 문구를 표시한다.
- 목록/알림/검색 결과에서는 쿠폰 번호와 바코드를 마스킹한다.
- 이미지 파일 검사 완료 전에는 바코드 열람을 허용하지 않는다.
- 유효기간이 지난 쿠폰은 경고 후 등록하거나 차단하는 정책을 일관되게 적용한다.

### F-02 사용 선점

인수 조건:

- `AVAILABLE` 상태에서 권한 있는 한 사람만 선점할 수 있다.
- 동시에 여러 요청이 와도 서버는 정확히 한 건만 성공시킨다.
- 기본 선점 시간은 15분이며 화면에 남은 시간을 표시한다.
- 다른 구성원은 선점자 표시 이름과 남은 시간만 확인한다.
- 선점자는 직접 해제할 수 있고 시간이 끝나면 자동 해제된다.

### F-03 바코드 열람과 사용 완료

인수 조건:

- 선점자가 재확인한 후에만 바코드/번호를 표시한다.
- 바코드 열람자와 시각을 감사 이력에 남긴다.
- 사용 완료 후 바코드를 다시 노출하지 않는다.
- 소유자/관리자는 사유를 입력해 잘못된 상태를 정정할 수 있다.
- 실제 발행사 사용 여부가 아니라 사용자가 표시한 상태임을 안내한다.

### F-04 만료 알림

인수 조건:

- 소유자가 30/14/7/1일 전 알림을 선택할 수 있다.
- 동일 시점 알림은 한 번만 발송한다.
- 만료일이 지나면 사용 가능/선점 중 쿠폰을 만료 처리한다.

## 12. 에픽 G — 알림과 설정

인수 조건:

- 앱 내 알림은 초대, 일정, 댓글, 쿠폰 상태를 지원한다.
- 이메일 알림은 유형별로 켜고 끌 수 있다.
- 사용자 시간대와 조용한 시간대를 적용한다.
- 알림 클릭 시 권한이 유지되는 경우 정확한 리소스로 이동한다.
- 권한이 회수된 알림은 민감한 제목/내용을 계속 노출하지 않는다.

## 13. 분석 이벤트

개인 내용, 쿠폰 번호, 글 본문은 분석 이벤트에 포함하지 않는다.

```text
sign_up_completed
space_created {space_type}
invitation_sent {channel}
invitation_accepted
event_created {is_recurring, visibility}
post_created {has_attachment}
coupon_created {days_until_expiry}
coupon_claim_succeeded
coupon_claim_conflicted
coupon_revealed
coupon_completed
coupon_claim_expired
```

## 14. 우선순위 백로그

### P0 — 비공개 베타 필수

- 인증, 개인 공간, 그룹/초대/역할
- 오늘 화면
- 일정 CRUD, 기본 반복, 참석, 알림
- 게시글/첨부/댓글
- 쿠폰 등록/선점/열람/완료/만료
- 앱 내 알림, 감사 이력, 계정 삭제
- 공간 격리·동시성·백업 복구 테스트

### P1 — 공개 베타 전 권장

- 소셜 로그인 2종 이상
- PWA 설치 안내와 Web Push
- 게시글/일정/쿠폰 통합 검색
- 쿠폰 상태 정정과 신고
- 일정에 자료 연결
- 사용량 한도와 운영자 지원 화면

### P2 — 사용성 검증 후

- 외부 캘린더 읽기 연동
- 쿠폰 OCR
- 네이티브 iOS·Android 앱
- 할 일/장보기 목록
- 실시간 문서 공동 편집

## 15. 출시 제외 기준

다음 중 하나라도 충족하면 베타 출시를 미룬다.

- 다른 그룹의 리소스 ID로 데이터가 조회됨
- 같은 쿠폰을 두 명이 동시에 선점할 수 있음
